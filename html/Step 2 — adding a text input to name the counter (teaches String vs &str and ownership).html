  Rust concepts: String vs &str                                                                                                                                            
                                                                                                                                                                           
  1. String in the struct                                                                                                                                                  
                                                                                                                                                                           
  struct CounterApp {                                                                                                                                                    
      name: String,  // owned, heap-allocated, growable                                                                                                                    
      count: i32,                                                                                                                                                          
  }                                                                                                                                                                        
                                                                                                                                                                           
  In Rust there are two main string types:                                                                                                                                 
   
  ┌────────┬─────────────┬──────────┬──────────┐                                                                                                                           
  │  Type  │   Owned?    │ Mutable? │  Where   │                                                                                                                         
  ├────────┼─────────────┼──────────┼──────────┤                                                                                                                           
  │ String │ Yes         │ Yes      │ Heap     │                                                                                                                         
  ├────────┼─────────────┼──────────┼──────────┤
  │ &str   │ No (borrow) │ No       │ Anywhere │
  └────────┴─────────────┴──────────┴──────────┘                                                                                                                           
   
  A struct must own its data, so we use String — not &str. If you tried name: &str, Rust would ask for a lifetime annotation (a more advanced topic).                      
                                                                                                                                                                         
  #[derive(Default)] gives String its default value of "" (empty string) automatically.                                                                                    
                                                                                                                                                                         
  ---                                                                                                                                                                      
  2. text_edit_singleline needs &mut String                                                                                                                              
                                                                                                                                                                           
  ui.text_edit_singleline(&mut self.name);                                                                                                                               
                                                                                                                                                                           
  &mut self.name is a mutable reference — we're lending the widget temporary write access to self.name. We don't give it ownership, just a borrow. egui reads and writes   
  through that reference each frame.                                                                                                                                       
                                                                                                                                                                           
  ---                                                                                                                                                                    
  3. .clone() vs .to_string()
                                                                                                                                                                           
  let display_name = if self.name.is_empty() {
      "Unnamed".to_string()   // &str  → new String                                                                                                                        
  } else {                                                                                                                                                                 
      self.name.clone()       // String → new String (copy the value)
  };                                                                                                                                                                       
                                                                                                                                                                         
  - "Unnamed" is a &str (string literal, baked into the binary). .to_string() converts it into an owned String.                                                            
  - .clone() makes a full copy of self.name so display_name owns its own data independently.
                                                                                                                                                                           
  You need the clone here because self.name is still owned by the struct — you can't move it out while self is still alive.
