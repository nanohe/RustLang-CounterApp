  Rust concepts introduced                                                                                                                                                 
                                                                                                                                                                           
  1. enum — a type with fixed variants                                                                                                                                     
                                                                                                                                                                           
  #[derive(Debug, Clone, Copy, PartialEq, Default)]                                                                                                                      
  enum Category {                                                                                                                                                          
      Daily,                                                                                                                                                               
      Weekly,                                                                                                                                                              
      #[default]                                                                                                                                                           
      Unlimited,                                                                                                                                                           
  }

  An enum defines a set of named possibilities. Unlike C enums, Rust enums are full types. The derives do real work:                                                       
  - Clone — lets you call .clone() on it
  - Copy — since it's tiny, Rust copies it automatically instead of moving (no need to .clone())                                                                           
  - PartialEq — lets you compare with == (needed by selectable_value)                                                                                                    
  - Debug — lets you print it with {:?}                                                                                                                                    
  - Default + #[default] — makes Unlimited the zero value, so CounterApp can still derive Default                                                                          
                                                                                                                                                                           
  ---                                                                                                                                                                      
  2. match — exhaustive pattern matching                                                                                                                                 
                                                                                                                                                                           
  fn label(&self) -> &str {                                                                                                                                              
      match self {                                                                                                                                                         
          Category::Daily => "Daily",                                                                                                                                    
          Category::Weekly => "Weekly",
          Category::Unlimited => "Unlimited",                                                                                                                              
      }
  }                                                                                                                                                                        
                                                                                                                                                                         
  match is like switch but Rust enforces you handle every variant. If you add a new variant later and forget to update a match, the compiler refuses to build. This is one 
  of Rust's most powerful safety features.
                                                                                                                                                                           
  ---                                                                                                                                                                    
  3. Option<T> — a value that may or may not exist

  fn max_count(&self) -> Option<i32> {
      match self {                                                                                                                                                         
          Category::Daily => Some(10),
          Category::Weekly => Some(50),                                                                                                                                    
          Category::Unlimited => None,                                                                                                                                   
      }                                                                                                                                                                    
  }
                                                                                                                                                                           
  Option<T> is Rust's replacement for null. It has two variants:                                                                                                         
  - Some(value) — there is a value
  - None — there isn't                                                                                                                                                     
                      
  You then use match (or if let) to safely unwrap it:                                                                                                                      
                                                                                                                                                                           
  match self.category.max_count() {
      Some(max) => self.count = new_count.min(max),  // cap at max                                                                                                         
      None => self.count = new_count,                 // no cap                                                                                                          
  }                                                                                                                                                                        
                  
  Rust forces you to handle both cases — you can never accidentally use a None as if it were a value (unlike null pointer bugs in other languages).
