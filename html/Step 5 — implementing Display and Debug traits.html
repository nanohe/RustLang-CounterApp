  Rust concepts introduced                                                                                                                                                 
                                                                                                                                                                           
  1. Traits — Rust's version of interfaces                                                                                                                                 
                                                                                                                                                                           
  A trait defines behaviour that a type must implement. Display and Debug are both traits from the standard library defined as:                                            
   
  trait Display {                                                                                                                                                          
      fn fmt(&self, f: &mut Formatter<'_>) -> fmt::Result;                                                                                                                 
  }                                                                                                                                                                        
                                                                                                                                                                           
  You implement a trait with impl TraitName for YourType { ... }. Any type that implements Display can be used with {} in format!, println!, .to_string(), etc.            
                                                                                                                                                                           
  ---                                                                                                                                                                      
  2. impl fmt::Display for Category                                                                                                                                      
                                                                                                                                                                           
  impl fmt::Display for Category {
      fn fmt(&self, f: &mut fmt::Formatter<'_>) -> fmt::Result {                                                                                                           
          let label = match self { ... };                                                                                                                                  
          write!(f, "{}", label)
      }                                                                                                                                                                    
  }                                                                                                                                                                      

  Once implemented, self.new_category.to_string() works automatically — the standard library provides .to_string() for free on any type that implements Display. We also   
  removed the label() method since Display now covers that role.
                                                                                                                                                                           
  ---                                                                                                                                                                    
  3. impl fmt::Display for Counter

  impl fmt::Display for Counter {
      fn fmt(&self, f: &mut fmt::Formatter<'_>) -> fmt::Result {                                                                                                           
          write!(f, "[{}] {}: {}{}", self.category, self.name, self.count, limit)
      }                                                                                                                                                                    
  }                                                                                                                                                                      
                                                                                                                                                                           
  Now the UI line shrinks from a verbose format!(...) to just:                                                                                                             
   
  ui.label(counter.to_string());                                                                                                                                           
                                                                                                                                                                           
  The formatting logic lives on the type, not scattered in the UI.                                                                                                         
                                                                                                                                                                           
  ---                                                                                                                                                                      
  4. Debug vs Display — two different traits for two audiences                                                                                                           
                                                                                                                                                                           
  ┌─────────┬────────┬───────────────────────────────────┐
  │  Trait  │ Format │              Purpose              │                                                                                                                 
  ├─────────┼────────┼───────────────────────────────────┤                                                                                                               
  │ Display │ {}     │ User-facing, you design it        │
  ├─────────┼────────┼───────────────────────────────────┤
  │ Debug   │ {:?}   │ Developer-facing, usually derived │                                                                                                                 
  └─────────┴────────┴───────────────────────────────────┘                                                                                                                 
                                                                                                                                                                           
  #[derive(Debug)]   // free implementation, shows raw field values                                                                                                        
  struct Counter { ... }                                                                                                                                                   
   
  The debug panel uses {:?}:                                                                                                                                               
  ui.monospace(format!("{:?}", counter));
  // → Counter { name: "steps", count: 3, category: Daily }                                                                                                                
                                                           
  Display shows what the user sees. Debug shows what you see when inspecting state — toggle the checkbox in the app to see both side by side.
