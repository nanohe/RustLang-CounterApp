  Rust concepts introduced                                                                                                                                                 
                                                                                                                                                                           
  1. External crates & Cargo.toml                                                                                                                                          
                                                                                                                                                                           
  serde = { version = "1", features = ["derive"] }                                                                                                                         
  serde_json = "1"                                                                                                                                                         
                                                                                                                                                                           
  Rust's package manager is Cargo. Crates are libraries. serde is Rust's de-facto serialization framework — it converts structs to/from formats like JSON, TOML, YAML. The 
  features = ["derive"] part enables the proc-macro that auto-generates serialization code.                                                                                
                                                                                                                                                                           
  ---             
  2. Derive macros for serialization

  #[derive(Serialize, Deserialize)]
  struct Counter { ... }
                                                                                                                                                                           
  Just like Debug and Default, adding these derives makes Counter automatically serializable. serde generates the conversion code at compile time — no runtime overhead.

  3. Result<T, E> and the ? operator
                                    
  fn save(counters: &[Counter]) -> Result<(), Box<dyn std::error::Error>> {
      let json = serde_json::to_string_pretty(counters)?;                                                                                                                  
      std::fs::write(SAVE_FILE, json)?;                                                                                                                                    
      Ok(())                                                                                                                                                               
  }                                                                                                                                                                        
                  
  Result<T, E> has two variants — Ok(value) for success, Err(error) for failure. It's Rust's alternative to exceptions.                                                    
   
  The ? operator is the key pattern: if the call returns Err, it immediately returns that error from the current function. Without ? you'd write:                          
                  
  let json = match serde_json::to_string_pretty(counters) {                                                                                                                
      Ok(v) => v,                                                                                                                                                          
      Err(e) => return Err(e.into()),
  };                                                                                                                                                                       
                  
  ? collapses that to one character. Chain as many fallible operations as you like cleanly.                                                                                
   
  Box<dyn std::error::Error> means "any error type" — serde_json errors and std::io errors are different types, but both implement the Error trait, so boxing them lets us 
  return either from the same function.
                                                                                                                                                                           
  ---             
  4. Handling Result at the call site
                                                                                                                                                                           
  self.status = match save(&self.counters) {
      Ok(()) => Some(format!("Saved to {}", SAVE_FILE)),                                                                                                                   
      Err(e) => Some(format!("Save error: {}", e)),                                                                                                                        
  };
                                                                                                                                                                           
  The UI always handles both outcomes explicitly. Errors show in the UI as a status message rather than crashing the app — this is the ergonomic Rust way.                 
   
  ---                                                                                                                                                                      
  5. &[Counter] — slice reference
                                                                                                                                                                           
  fn save(counters: &[Counter]) -> ...
                                                                                                                                                                           
  Instead of taking &Vec<Counter>, we take &[Counter] (a slice). A slice is a view into any contiguous sequence. This is more general — the function works on any slice,   
  not just Vec. It's idiomatic Rust to prefer &[T] over &Vec<T> in function parameters.
